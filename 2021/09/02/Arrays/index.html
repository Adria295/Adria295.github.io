Interfaces

  • Java has a method called Arrays.sort (in java.utils.Arrays)
  • It knows how to compare its own primitives and objects already

How do we tell Java how to compare our objects?

  • An Interface defines a set of methods

  • If a class implements an interface, it must implement the methods in the interface

  • One such interface is Comparable which includes a single method compareTo

  • If a class implements Comparable, we can use Arrays.sort

    • I.e. Java knows that our class will have a compareTo method, and this is what Arrays.sort uses.

●Compare with the fact that our sort method didn’t care about anything in the Student class except that it had a getGrade() method.

Summary

  • Try and avoid paired arrays

    • It will almost certainly lead to errors

  • If you are faced with a problem that you think requires paired arrays, it is almost certainly better to make your own class

  • Arrays of your own class can be sorted using Java’s Arrays.sort() method as long as your class implements the Comparable interface.

    • Which allows Java to compare two of your objects, and decide which is bigger, and which is smaller, and therefore sort them.
