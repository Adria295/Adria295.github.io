Functional Dependency Theory

Roadmap

  1. Functional Dependency Theory for quantifying the degree of goodness.（量化好坏）

  2. Normalization Theory for transforming a relational schema into a set of good and efficient relations.

  3. Transforming any relation to the Boyce-Codd Normal Form (BCNF) dealing with fictitious tuples.（虚构）

GUIDELINES FOR A GOOD DESIGN

Guideline 1: The attributes of a relation should make sense

  1. Attributes of different entities, e.g., students, employees, courses,should not be in the same relation.

    （尽量减少关系间的相似性，例如学生表不出现课程的属性？）

  2. Any relationship between relations should be represented only through Foreign Keys & Primary Keys

Guideline 2: Avoid redundant tuples

Guideline 3: Relations should have as a few NULL values as possible:

Statistics: Attributes that are frequently NULL should be placed in separate relations to avoid wasting storage & reducing uncertainty!

（常为空属性可以放进单独的relation里来节省空间）

Guideline 4: Design relations to avoid fictitious tuples after join

Theory

Functional Dependency (FD) is a formal metric of the degree of goodness of relational schema

（FD时关系模型好坏的度量）

In other words: give me a value of X and I’ll tell you which is the value of Y in a specific tuple! (X uniquely determines Y)

If t1[X] = t2[X] then t1[Y] = t2[Y] （相同的X，必有相同的Y）

Lemma 1: If K is a Candidate Key, then K functionally determines all attributes in relation R.

(CK表示最小的唯一键集合，所以决定了所有属性)

Lemma 2:

  • (Reflexive) If Y ⊆ X then X → Y e.g. X = {SSN, Ename}; X → {SSN}, X → {Ename}
  • (Augmentation) If X → Y then X U {Z} → Y U {Z}
  • (Transitive) If X → Y and Y → Z then X → Z

NORMALIZATION

Algorithm:

  1. Assert which are the FDs among attributes (no other semantics required)
  2. Take a pool and put into all the asserted FDs.
  3. Create a universal big relation with all attributes
  4. Recursively decompose the big relation based on the FDs into many smaller ones, such that, when we re-join them, it guarantees that no information is lost and re-constructs the original big relation without fictitious tuples.

This is the normalization process.

THEORY OF NORMALIZATION

Progressive decomposition of unsatisfactory (bad) relations by breaking up their attributes into smaller good relations;

(把不好的关系分解成小的好的关系叫做规范化)

  • A prime attribute is an attribute that belongs to some candidate key of the relation
  • A non-prime attribute is not a prime attribute, i.e., it is not a member of any candidate key

FIRST NORMAL FORM (1NF)

  • 1NF disallows:
    • nested attributes,
    • multivalued attributes

Note: a relation in 1NF is expected to have redundant & repeated values（允许冗余数据）

SECOND NORMAL FORM (2NF)

full FD X → Y means that if we remove any prime attribute A from the primary key X, then:X 去掉A不能再determineY。

（{SSN, Pnumber} → Ename is not a full FD (partial dependency), since SSN → Ename holds true;）

Definition: A relation R is in 2NF if every non-prime attribute A in R is fully functionally dependent on the primary key of R.

（每一个non-prime 必须由prime一起决定，单独就能决定的话就是PD）
