Algorithms and Complexity

Array Data Structure

  • An array is a sequence of slots. Each slot contains an element (value or object). Each slot has a fixed index.

Insertion

  1. Copy a[ins…right–1] into a[ins+1…right].
  2. Copy elem into a[ins].
  3. Terminate.

Time complexity is O(n).

Deletion

  1. Copy a[del+1…right] into a[del…right–1].
  2. Make a[right] unoccupied.
  3. Terminate.

Time complexity is O(n)

Searching

Linear search

time complexity is O(n)

Binary search

Merging（有序）

  1. Set i = l1, set j = l2, and set k = l3.
  2. While i £ r1 and j £ r2, repeat:
    2.1. If a1[i] is less than or equal to a2[j]:
    2.1.1. Copy a1[i] into a3[k], then increment i and k.
    2.2. Else:
    2.2.1. Copy a2[j] into a3[k], then increment j and k.
  3. If i £ r1, copy a1[i…r1] into a3[k…r3].
  4. If j £ r2, copy a2[j…r2] into a3[k…r3].
  5. Terminate.
  • No. of copies = n1 + n2 = n Time complexity is O(n).
  • Max. no. of comparisons = n – 1 Time complexity is again O(n).

sort

1.Selection sort （右边选最小）

  1. For l = left, …, right–1, repeat:
    1.1. Set p such that a[p] is the least 
               of *a*[*l*…*right*].
    1.2. If p ！=l, swap a[p] and a[l].
  2. Terminate.

No. of comparisons = *n^2/2

Time complexity is O(n^2).

2.Insertion sort（理扑克牌 把右边牌插空到左边已经理好的牌中）

3.Merge-sort

comps(n) » 2 comps(n/2) + n – 1

comps(n) » n log2n

Time complexity is O(n log n)

Space complexity is O(n), since step 1.4 needs an auxiliary array of length n.

4.Quick-sort

  1. If left < right:
    1.1. Partition a[left…right] such that
    a[left…p–1] are all less than or equal to a[p], and
    a[p+1…right] are all greater than or equal to a[p].
    1.2. Sort a[left…p–1] into ascending order.
    1.3. Sort a[p+1…right] into ascending order.
  2. Terminate.

Best-case time complexity is O(n log n). Worst-case time complexity is O(n^2).

Quick-sort partitioning :

  1. Let pivot be the element in a[left], and set p = left.
  2. For r = left+1, …, right, repeat:
    2.1. If a[r] is less than pivot:
    2.1.1. Copy a[r] into a[p], a[p+1] into a[r],
    and pivot into a[p+1].
    2.1.2. Increment p.
  3. Terminate yielding p.

Linked-List Data Structures

  • A linked-list consists of a header together with a sequence of nodes connected by links:

singly-linked-list (SLL)

1.traversal

curr = curr.next；

2.delete

first = first.next;

3.insertion

  1. Make ins a link to a newly-created node with element elem
    and next node null.
  2. If the insertion point is before the first node:
    2.1. Set ins’s next node to first.
    2.2. Set first to ins.
  3. Else, if the insertion point is after the node pred:
    3.1. Set ins’s next node to pred’s next node.
    3.2. Set pred’s next node to ins.
  4. Terminate.

4.Deletion

To delete node del from the SLL headed by first:

  1. Let next be del’s next node.
  2. If del = first:
    2.1. Set first to next.
  3. Else:
    3.1. Let pred be del’s predecessor.
    3.2. Set pred’s next to next.
  4. Terminate.

Time complexity is O(n).

5.Searching

time complexity is O(n)

doubly-linked-list (DLL)

Algorithm SLL DLL
Insertion O(1) O(1)
Deletion O(n) O(1)
Search O(n) O(n)

删除的不同原因在于，SLL不知道他的pre，所以只能一个个去找node，直到某一个node的next是del。但是DLL知道他的pre，所以O（1）

Stack ADTs

  • A stack is a last-in-first-out sequence of elements. （堆书）

•Requirements:

1)It must be possible to make a stack empty.

2)It must be possible to push an element on to a stack (i.e., add it at the top of the stack).

3)It must be possible to pop the topmost element from a stack (i.e., remove it from the stack).

4)It must be possible to test whether a stack is empty.

5)It should be possible to access the topmost element in a stack without popping it.

实现方式：array or SLL

Queue ADTs

  • A queue is a first-in-first-out sequence of elements（排队等车）

•Requirements:

1)It must be possible to make a queue empty.

2)It must be possible to test whether a queue is empty.

3)It must be possible to obtain the size of a queue.（比stack多一个）

4)It must be possible to add an element at the rear of a queue.

5)It must be possible to remove the front element from a queue.

6)It must be possible to access the front element in a queue without removing it.

用array实现，当右边满了的时候，addlast是O(n), rather than O(1). 所以可以使用cyclic array

List and Iterator ADTS

•Requirements:

1)It must be possible to make a list empty.

2)It must be possible to test whether a list is empty.

3)It must be possible to obtain the length of a list.

4)It must be possible to add an element at any position in a list.

5)It must be possible to remove the element at any position in a list.

6)It must be possible to inspect or update the element at any position in a list.

7)It must be possible to concatenate lists.

8)It must be possible to test lists for equality.

9)It must be possible to traverse a list.

… list.get(p) …
… list.set(p, x) …

•But this traversal could have time complexity O(n2), if get and set turn out to be O(n).

2.Iterator:

Binary-Search-Tree Data Structure

  • A binary-tree consists of a header together with a number of nodes connected as follows:（二叉树）

  • A binary-tree of depth d is balanced if all nodes at depths 0, 1, …, d–2 have two children.

  • A binary-search-tree (or BST) is a binary-tree in which ：（二搜树）

    •Let elem be the element contained in the node.

    •The node’s left subtree (if non-empty) contains only elements less than elem.

    •The node’s right subtree (if non-empty) contains only elements greater than elem.

  • A balanced BST of depth d has at least 2d and at most 2d+1–1 nodes.

BST search

•If the BST is balanced, d » log n

Best-case time complexity is O(log n).

•If the BST is unbalanced, d » n:

Worst-case time complexity is O(n).

BST insertion

Search and replace null link by a link to the new element.

时间复杂度同search

BST deletion

Case D (topmost node has two children):
Copy the right subtree’s leftmost element into the topmost node, then delete the right subtree’s leftmost element. ！！！

Binary-tree traversal

•In-order traversal: Traverse the left subtree, then visit the root node, then traverse the right subtree.

•Pre-order traversal: Visit the root node, then traverse the left subtree, then traverse the right subtree.

•Post-order traversal: Traverse the left subtree, then traverse the right subtree, then visit the root node.

AVL Trees

Balanced Binary Search Trees (AVL Trees)

Height of an empty tree is -1

Height of a single node is 0

Seacrh Deletion相似于BST，Insert完需要保持平衡

AVL平衡条件与BST不同，BST：d-2层有两个child

AVL：左右子树深度相差不超过1

right rotation

自己成为left child的 right ，然后让left child的right child变成自己的left。

四种情况：1.left left ：right

2.right right ： left

3.left subtree of right child: right left 第一次是right节点为中心来转

4.right subtree of left child:left right

Map ADTs

•A map is a collection of entries, in no fixed order, in which all entries have distinct keys.

•Each entry is a tuple that consists of a key field and a value field.

Hash-Table Data Structures

•A hash-table is an array of m buckets, together with a hash function hash(k) that translates each key k to a bucket index (in range 0…m–1).

•Consider m = 9,973, hash(k) = k modulo m.

+ Good number of buckets. Load factor » 0.6. m通过load factor决定num of buckets

+ Even distribution (since m is prime). hash的方法决定分配是否均匀
